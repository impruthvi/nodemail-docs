# Changelog

All notable changes to `@impruthvi/nodemail` are documented here.

---

## v1.1.3

**CLI fix**

- Fixed `queue:clear` CLI command to use `--status/-s` option instead of positional argument
- CLI now matches documented usage: `npx nodemail queue:clear --status failed`

---

## v1.1.2

**Queue compatibility fix**

- **queue**: Replace colon with underscore in queue names for BullMQ compatibility
  - BullMQ does not allow colons (`:`) in queue names
  - Queue names now formatted as `nodemail_mail` instead of `nodemail:mail`

---

## v1.1.1

**Bug fixes and test improvements**

- Type handling and error management in preview and queue commands
- Improved async/sync handling in image embedding and priority tests
- Added comprehensive test coverage for MailFacade, MailManager, Mailable, and QueueManager

---

## v1.1.0

**CLI Commands** ‚Äî powerful command-line tools for managing your email workflow.

- **[CLI Commands](/docs/cli)** ‚Äî New CLI tool with queue management, email preview, code generation, and configuration validation.
  - `queue:work` ‚Äî Start processing queued emails
  - `queue:status` ‚Äî Show queue job counts and status
  - `queue:clear` ‚Äî Clear jobs from the queue by status
  - `queue:retry` ‚Äî Retry failed jobs in the queue
  - `preview` ‚Äî Preview an email in your browser without sending
  - `send:test` ‚Äî Send a test email to verify configuration
  - `make:mailable` ‚Äî Generate a new Mailable class
  - `config:check` ‚Äî Validate your mail configuration

---

## v1.0.1

**5 new features** expanding the library's email capabilities.

- **[Email Events](/docs/email-events)** ‚Äî Listen to `sending`, `sent`, and `failed` lifecycle events with typed listeners.
- **[Email Preview](/docs/email-preview)** ‚Äî Render emails in the browser during development without actually sending them.
- **[Email Priority](/docs/email-priority)** ‚Äî Set priority headers (`high`, `normal`, `low`) on outgoing messages.
- **[Embedded Images](/docs/embedded-images)** ‚Äî Attach inline images via CID references for rich HTML emails.
- **[Rate Limiting](/docs/rate-limiting)** ‚Äî Throttle outgoing email with per-provider rate limits and configurable strategies.

---

## v1.0.0

**Provider Failover** ‚Äî automatic failover chains with retries, delays, and callbacks.

- `FailoverManager` class for managing provider chains
- `failover` config option (global and per-mailer)
- `onFailover` callback for monitoring provider switches
- `MailResponse` now includes `provider`, `failoverUsed`, `failoverAttempts`
- Testing utilities: `MailFake.simulateFailures()` / `resetFailures()`
- `AssertableMessage` failover methods: `wasFailoverUsed()`, `getProvider()`, `getFailoverAttempts()`, `getResponse()`

See the [Provider Failover](/docs/failover) docs and [Migration Guide](/docs/migration) for adoption details.
